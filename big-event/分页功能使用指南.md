## 🔧 技术选型

本项目使用 **PageHelper** 作为分页插件，它是 MyBatis 的一个分页插件，使用简单，功能强大。

### 依赖配置

在 `pom.xml` 中添加以下依赖：

```xml
<!-- PageHelper 分页插件 -->
<dependency>
    <groupId>com.github.pagehelper</groupId>
    <artifactId>pagehelper-spring-boot-starter</artifactId>
    <version>1.4.7</version>
</dependency>
```

## 💻 完整代码示例

### 1. PageBean.java - 分页结果封装类

```java
package com.itheima.pojo;

import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;
import java.util.List;

/**
 * 分页返回结果对象
 * @param <T> 泛型，可以是任何实体类
 */
@Data
@NoArgsConstructor      // 无参构造
@AllArgsConstructor     // 有参构造
public class PageBean<T> {
    private Long total;      // 总记录数
    private List<T> items;   // 当前页数据集合
}
```

**说明：**
- `total`: 查询结果的总记录数
- `items`: 当前页的数据列表
- 使用泛型 `<T>` 可以适用于任何实体类

---

### 2. ArticleMapper.java - Mapper接口

```java
package com.itheima.mapper;

import com.itheima.pojo.Article;
import org.apache.ibatis.annotations.Mapper;
import java.util.List;

@Mapper
public interface ArticleMapper {
    /**
     * 分页查询
     * 注意：这里不需要传入 pageNum 和 pageSize
     * PageHelper 会自动处理分页逻辑
     */
    List<Article> list(Integer userId, Integer categoryId, String state);
}
```

**关键点：**
- Mapper 方法**不需要**接收 `pageNum` 和 `pageSize` 参数
- 返回类型是 `List<T>`，不是 `PageBean`
- PageHelper 会在底层自动拦截并添加分页 SQL

---

### 3. ArticleMapper.xml - MyBatis映射文件

```xml
<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="com.itheima.mapper.ArticleMapper">
    <!-- 动态SQL查询 -->
    <select id="list" resultType="com.itheima.pojo.Article">
        select * from article
        <where>
            <if test="categoryId!=null">
                category_id=#{categoryId}
            </if>
            <if test="state!=null">
                and state=#{state}
            </if>
            and create_user=#{userId}
        </where>
    </select>
</mapper>
```

**说明：**
- SQL 语句**不需要**写 `LIMIT` 子句
- PageHelper 会自动在 SQL 后面添加 `LIMIT` 语句

### 5. ArticleServiceImpl.java - Service实现类（核心）

```java
package com.itheima.service.impl;

import com.github.pagehelper.Page;
import com.github.pagehelper.PageHelper;
import com.itheima.mapper.ArticleMapper;
import com.itheima.pojo.Article;
import com.itheima.pojo.PageBean;
import com.itheima.service.ArticleService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;
import java.util.List;

@Service
public class ArticleServiceImpl implements ArticleService {
    
    @Autowired
    private ArticleMapper articleMapper;
    
    @Override
    public PageBean<Article> list(Integer pageNum, Integer pageSize, 
                                  Integer categoryId, String state) {
        // 步骤1: 创建 PageBean 对象
        PageBean<Article> pb = new PageBean<>();
        
        // 步骤2: 开启分页查询（必须在调用 Mapper 方法之前）
        PageHelper.startPage(pageNum, pageSize);
        
        // 步骤3: 调用 Mapper 方法查询数据
        List<Article> as = articleMapper.list(userId, categoryId, state);
        
        // 步骤4: 将 List 转换为 Page 对象（PageHelper 会自动转换）
        Page<Article> p = (Page<Article>) as;
        
        // 步骤5: 填充 PageBean 对象
        pb.setTotal(p.getTotal());      // 设置总记录数
        pb.setItems(p.getResult());     // 设置当前页数据
        
        return pb;
    }
}
```

**核心步骤说明：**

1. **创建 PageBean 对象**：用于封装返回结果
2. **开启分页**：`PageHelper.startPage(pageNum, pageSize)` 
   - ⚠️ **必须在调用 Mapper 方法之前调用**
   - `pageNum`: 页码，从 1 开始
   - `pageSize`: 每页记录数
3. **调用 Mapper**：执行查询，此时 PageHelper 会自动拦截并添加分页 SQL
4. **类型转换**：将返回的 `List` 转换为 `Page` 对象
5. **提取数据**：
   - `p.getTotal()`: 获取总记录数
   - `p.getResult()`: 获取当前页的数据列表

### 6. ArticleController.java - Controller层

```java
package com.itheima.controller;

import com.itheima.pojo.Article;
import com.itheima.pojo.PageBean;
import com.itheima.pojo.Result;
import com.itheima.service.ArticleService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/article")
public class ArticleController {
    
    @Autowired
    private ArticleService articleService;
    
    /**
     * 分页查询接口
     * @param pageNum 页码（从1开始）
     * @param pageSize 每页记录数
     * @param categoryId 分类ID（可选）
     * @param state 状态（可选）
     * @return 分页结果
     */
    @GetMapping("/list")
    public Result<PageBean<Article>> list(
            Integer pageNum,                    // 必传参数
            Integer pageSize,                   // 必传参数
            @RequestParam(required = false) Integer categoryId,  // 可选参数
            @RequestParam(required = false) String state         // 可选参数
    ) {
        PageBean<Article> pb = articleService.list(pageNum, pageSize, categoryId, state);
        return Result.success(pb);
    }
}
```

**请求示例：**
```
GET /article/list?pageNum=1&pageSize=10&categoryId=1&state=已发布
```

**响应示例：**
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "total": 100,        // 总记录数
    "items": [           // 当前页数据
      {
        "id": 1,
        "title": "文章标题",
        "content": "文章内容",
        ...
      },
      ...
    ]
  }
}
```

## 📝 编写流程

### 快速实现分页功能的 5 个步骤：

#### 步骤 1: 添加依赖（如果还没有）
在 `pom.xml` 中添加 PageHelper 依赖

#### 步骤 2: 创建 PageBean 类
创建通用的分页结果封装类（如果项目中还没有）

#### 步骤 3: 编写 Mapper 接口和 XML
- Mapper 接口方法：返回 `List<T>`，**不接收** `pageNum` 和 `pageSize`
- XML 中的 SQL：**不写** `LIMIT` 子句

#### 步骤 4: 编写 Service 层
```java
public PageBean<T> list(Integer pageNum, Integer pageSize, ...) {
    PageBean<T> pb = new PageBean<>();
    PageHelper.startPage(pageNum, pageSize);  // 开启分页
    List<T> list = mapper.list(...);          // 调用 Mapper
    Page<T> p = (Page<T>) list;               // 类型转换
    pb.setTotal(p.getTotal());                 // 设置总数
    pb.setItems(p.getResult());                // 设置数据
    return pb;
}
```

#### 步骤 5: 编写 Controller 层
接收 `pageNum` 和 `pageSize` 参数，调用 Service 方法

## ⚠️ 注意事项

### 1. PageHelper.startPage() 的位置
- ✅ **正确**：必须在调用 Mapper 方法**之前**调用
- ❌ **错误**：在调用 Mapper 方法之后调用（不会生效）

```java
// ✅ 正确示例
PageHelper.startPage(pageNum, pageSize);
List<Article> list = articleMapper.list(...);

// ❌ 错误示例
List<Article> list = articleMapper.list(...);
PageHelper.startPage(pageNum, pageSize);  // 太晚了，不会生效
```

### 2. 页码从 1 开始
- `pageNum` 的值从 **1** 开始，不是 0
- 如果传入 0 或负数，PageHelper 会使用默认值

### 3. 线程安全
- PageHelper 使用 `ThreadLocal` 存储分页参数
- 分页参数只对**紧接着**的第一次查询有效
- 查询完成后会自动清除 ThreadLocal，避免影响其他查询

### 4. 多个查询的情况
如果在一个方法中有多个查询，需要为每个查询单独开启分页：

```java
// 查询1
PageHelper.startPage(pageNum1, pageSize1);
List<Article> list1 = articleMapper.list1(...);

// 查询2（需要重新开启分页）
PageHelper.startPage(pageNum2, pageSize2);
List<Category> list2 = categoryMapper.list2(...);
```

### 5. 不需要手动写 LIMIT
- ❌ **不要**在 SQL 中手动写 `LIMIT` 子句
- ✅ PageHelper 会自动添加 `LIMIT` 语句

### 6. 返回类型
- Mapper 方法返回 `List<T>`，不是 `PageBean<T>`
- PageHelper 会将 `List` 转换为 `Page` 对象（继承自 `ArrayList`）

## 📊 分页原理

PageHelper 的工作原理：

1. **拦截器机制**：PageHelper 通过 MyBatis 拦截器拦截 SQL 语句
2. **自动添加 LIMIT**：在原始 SQL 后面自动添加 `LIMIT` 子句
3. **执行 COUNT 查询**：自动执行 `SELECT COUNT(*)` 获取总记录数
4. **封装结果**：将查询结果封装到 `Page` 对象中

**示例：**

原始 SQL：
```sql
SELECT * FROM article WHERE create_user = 1
```

PageHelper 自动转换后：
```sql
-- 1. 查询总数
SELECT COUNT(*) FROM article WHERE create_user = 1

-- 2. 查询分页数据
SELECT * FROM article WHERE create_user = 1 LIMIT 0, 10
```

## 🎯 总结

### 核心要点记忆：

1. **依赖**：`pagehelper-spring-boot-starter`
2. **开启分页**：`PageHelper.startPage(pageNum, pageSize)` 必须在 Mapper 调用之前
3. **Mapper 方法**：返回 `List<T>`，不接收分页参数
4. **SQL 语句**：不写 `LIMIT`，PageHelper 自动添加
5. **结果封装**：使用 `PageBean<T>` 封装返回结果

### 快速记忆口诀：
> **依赖 → PageBean → Mapper（不写LIMIT）→ Service（startPage + 查询 + 转换）→ Controller（接收参数）**
